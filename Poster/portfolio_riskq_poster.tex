\begin{block}{Introduction}
Portfolio risk requires accurate tail estimates such as VaR and CVaR, yet production systems must balance accuracy with compute cost. Portfolio RiskQ explores a classical-to-quantum workflow that preserves interpretability while opening a path to faster tail estimation.
\end{block}

\begin{block}{Condensed Abstract}
We built an end-to-end pipeline to estimate portfolio tail risk, spanning data calibration, stress scenarios,

nested Monte Carlo, discretization into $2^n$ bins, and an IQAE simulator that estimates tail probability with confidence intervals.
\end{block}

\begin{block}{Background}
VaR and CVaR summarize the loss tail that matters most to risk teams, while nested Monte Carlo provides a robust classical baseline. Quantum amplitude estimation promises a quadratic reduction in sampling complexity, but it requires discretized, quantum-ready distributions.
\vspace{0.3em}
Think of VaR as a “loss line you rarely cross,” and CVaR as the “average loss when you do cross it.”
\vspace{0.3em}
In plain language, we simulate many possible futures, summarize the worst losses, and compare a classical estimate with a quantum-inspired estimate of the same tail risk.
\vspace{0.3em}
We treat returns as random and generate many loss outcomes for the portfolio.
\vspace{0.3em}
Intuition: the quantum step is a smarter way to estimate a probability (the tail) with fewer effective samples.
\vspace{0.3em}
Reading the example: the tail probability is about 0.83\%, and the interval shows expected uncertainty.
\vspace{0.3em}
This makes the classical vs quantum comparison fair and easy to explain to non-specialists.
\end{block}

\begin{block}{Hypothesis}
IQAE-style estimation on discretized loss distributions can approximate tail probability with fewer effective samples while preserving interpretability for risk stakeholders.
\end{block}

\begin{block}{Objectives}
\begin{itemize}
    \item Build a working classical risk engine and dashboard MVP.
    \item Add discretization and tail-mask generation.

\vspace{0.3em}
The emphasis is on a complete, inspectable pipeline that supports both classical validation and quantum-oriented estimation.
\end{block}

\begin{block}{Methodology}
\textbf{Nested Monte Carlo (loss samples)}
\begin{align*}
    r \sim \mathcal{N}(\mu,\Sigma), \quad L = -w^\top r
\end{align*}
\textbf{Tail metrics}
\begin{align*}
    \text{VaR}_{\alpha} &= Q_{\alpha}(L) \\
    \text{CVaR}_{\alpha} &= \mathbb{E}[L \mid L \ge \text{VaR}_{\alpha}]
\end{align*}
\vspace{0.4em}
\textbf{Discretization}
\begin{itemize}
    \item Bin losses into $2^n$ intervals.
    \item Produce edges, probabilities, tail mask.
    \item Bootstrap error estimate for bin stability.
\end{itemize}
\vspace{0.4em}
\textbf{IQAE (simulator)}
\begin{align*}
    p &= \sin^2(\theta), \quad p_m = \sin^2((2m+1)\theta)
\end{align*}
\begin{itemize}
    \item Uses Grover iteration schedule to amplify tail states.
    \item Returns estimate with confidence interval.
\end{itemize}
\end{block}

\begin{block}{Analysis}
\begin{itemize}
    \item Validate probability normalization and tail-mask correctness.
    \item Compare IQAE estimates to true tail probability on toy data.
    \item Track resource usage (qubits, depth, 2Q gates, oracle calls).
    \item Compare classical vs quantum using same $\ell$ and histogram.
\end{itemize}
\vspace{0.3em}
All comparisons share the same tail threshold $\ell$, scenario configuration, and histogram to ensure a fair baseline.
\end{block}

\begin{block}{Discussion}
\begin{itemize}
    \item Discretization provides a stable bridge to IQAE.
    \item Simulator results align with expected tail probabilities.
    \item Stress sliders need full backend integration.
    \item Comparison uses same threshold $\ell$, scenario, and histogram.
    \item Padding can affect tail resolution at small bin counts.
\end{itemize}
\vspace{0.3em}
Padding effects are explicitly tracked to avoid overstating precision at small bin counts.
\end{block}
